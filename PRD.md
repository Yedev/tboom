# T-Boom 产品需求文档 (PRD)

## 1. 产品概述

T-Boom 是一款融合俄罗斯方块与平台跳跃的创新混合游戏。玩家同时操控两个维度：下落的俄罗斯方块和棋盘内的角色。角色可以在已锁定的方块上奔跑跳跃，使用炸弹破坏方块，但必须躲避方块碾压和爆炸伤害。

**核心卖点**：俄罗斯方块 + 平台跳跃 + 炸弹破坏 + 双面卡牌构建 + BOSS挑战 + 分支路线 = 多重策略博弈

## 2. 目标用户

- 休闲手游玩家（竖屏操作）
- 喜欢创新玩法的俄罗斯方块爱好者
- 喜欢像素风格的独立游戏玩家

## 3. 平台与技术

| 项目 | 规格 |
|------|------|
| 框架 | Phaser 3 + TypeScript + Vite |
| 画布 | 520 x 960（竖屏） |
| 平台 | Web（桌面键盘 + 移动触屏） |
| 渲染 | WebGL 优先，Canvas 回退 |
| 缩放 | Phaser.Scale.FIT + CENTER_BOTH |
| 音频 | Phaser Web Audio |

## 4. 游戏机制

### 4.1 俄罗斯方块系统

- **棋盘**：10列 x 20行，标准尺寸
- **方块**：7种标准方块（I/J/L/O/S/T/Z），4个旋转状态
- **旋转**：SRS（Super Rotation System）标准踢墙数据
- **跟随模式**：新方块跟随角色X位置自动移动，方块同时受重力下落
- **确认放置**：按 W 或触摸「放置」按钮，方块停止跟随，从当前位置继续下落
- **硬降**：按 S 或触摸「速降」按钮，方块瞬间落到底部
- **锁定延迟**：500ms，最多15次移动延长锁定
- **消行**：满行消除，400ms闪烁动画
- **计分**：1行=100, 2行=300, 3行=500, 4行=800 × 当前得分倍率
- **下落速度**：800ms起步，每级-40ms，最低50ms

### 4.2 关卡进度系统

游戏以"关卡（Stage）"为单位推进，每关有固定消行目标：

- **目标行数**：`5 + (关卡-1) × 3`（第1关=5行，第2关=8行，第3关=11行，以此类推）
- **关卡完成**：累计消行达到目标后进入 `STAGE_CLEAR` 状态
- **关卡奖励**：关卡完成后展示升级卡选择界面，玩家选择一项永久强化
- **关卡重置**：进入新关卡时，本关消行计数清零，棋盘和角色状态保留

### 4.3 角色系统

- **尺寸**：20 x 24 像素
- **生命值**：初始 5 HP（可通过升级增加），用心形显示
- **移动**：左右移动基础 160 px/s（可通过升级提高）
- **跳跃**：初速度 -420 px/s（向上），重力 1200 px/s²（可通过升级提高跳跃高度）
- **碰撞**：AABB 碰撞检测，分轴解算（先X后Y）
- **碾压伤害**：方块锁定时覆盖角色位置 → 扣2 HP + 推到最近安全位置
- **无敌帧**：受伤后1500ms无敌，闪烁效果
- **死亡条件**：HP=0 或 角色掉出棋盘底部
- **动画**：待机（呼吸压缩）、行走（腿部交替）、跳跃（收腿举臂）、下落（展开）、受伤（红色+X眼）

### 4.4 炸弹系统

- **获取**：每锁定一个方块获得 +1 炸弹，初始上限3个（可通过升级增加）
- **放置**：角色脚下生成，受重力影响（800 px/s²）
- **引信**：2000ms，在 1200/800/400ms 时发出滴答声
- **爆炸半径**：
  - 破坏半径：1格（3x3 范围内方块被摧毁，可通过升级扩大）
  - 伤害半径：2格（角色扣2 HP）
- **提前引爆**：下落的俄罗斯方块碰到炸弹立即引爆
- **视觉**：爆炸 400ms 橙黄色闪烁

### 4.5 强化卡牌系统

每完成一个普通关卡，玩家从3张随机卡中选择1张。卡牌效果在本局游戏内永久累积叠加。

#### 设计原则：双面效果 × 得分行为拆分

每张卡牌都有**正面效果（Positive）**和**负面效果（Negative）**，玩家需要权衡取舍来构建自己的 build。

负面效果的设计围绕**得分行为拆分**展开——游戏中的得分来源分为以下几类：
- **消除方块**（消行得分）
- **炸弹摧毁方块**（炸弹得分）
- **击败史莱姆**（击杀得分）
- **特殊方块奖励**（见 §4.7）

每张卡的正面效果增强某一维度，负面效果则削弱其他得分维度，形成策略取舍。

#### 卡牌列表（初版设计）

| 卡牌ID | 名称 | 正面效果（P） | 负面效果（N） | 得分影响 |
|--------|------|-------------|-------------|---------|
| double_jump | 二段跳 | 角色可二段跳跃 | 史莱姆跳跃高度 +50% | 击杀史莱姆更难 |
| jump_boost | 弹簧腿 | 跳跃高度 +25% | 消行得分 -15% | 消行收益降低 |
| hp_boost | 铁甲心 | 最大血量 +2 | 炸弹破坏得分 -20% | 炸弹收益降低 |
| speed_boost | 疾步者 | 移动速度 +20% | 史莱姆移动速度 +20% | 击杀难度上升 |
| bomb_radius | 烈焰弹 | 炸弹破坏范围 +1格 | 炸弹自伤范围同步扩大 | 炸弹使用风险上升 |
| bomb_capacity | 弹药库 | 炸弹持有上限 +1 | 每关目标行数 +2 | 通关压力增加 |
| score_multiplier | 财神附体 | 消行得分倍率 +50% | 每块方块锁定补给炸弹概率 -30% | 炸弹资源减少 |
| slime_loot | 猎人之魂 | 击杀史莱姆额外掉落炸弹 | 史莱姆数量上限 +1 | 压力与收益并存 |
| line_clear_boost | 清道夫 | 同时消多行时额外加分 | 炸弹爆炸不再摧毁方块（仅伤害） | 炸弹摧毁得分归零 |

**卡牌UI**：半透明黑色遮罩 + 3张竖排卡片，正面效果（绿色）和负面效果（红色）分区显示；鼠标悬停高亮，点击/触碰选择。

### 4.6 特殊方块系统

除标准方块外，棋盘中会随机出现特殊类型的方块，增加策略深度。

| 方块类型 | 外观 | 特性 |
|---------|------|------|
| 普通方块 | 标准颜色 | 默认行为 |
| 硬化方块 | 深灰色+纹理 | 炸弹无法摧毁，只能消行消除 |
| 宝箱方块 | 金色+星形 | 被炸弹摧毁或消行消除时：随机奖励（+1血量 / +1炸弹 / +50分 / 得分倍率临时提升） |
| 毒方块 | 紫色+骷髅 | 被消行消除时扣1 HP，被炸弹摧毁则安全 | 
| 加速方块 | 橙色+箭头 | 锁定后本关方块下落速度永久提升一档 |
| 炸弹方块 | 红色+导火索 | 下落碰到其他锁定方块时自动爆炸（3x3范围） |

**特殊方块生成规则**：
- 第1关只生成普通方块，第2关起随机混入特殊方块
- 每关特殊方块占比上限：`min(10% × 关卡数, 40%)`
- 硬化方块不参与消行计分（消除时分数为0）

### 4.7 BOSS关卡系统

每隔3个普通关卡出现一个 BOSS 关（第4、7、10…关），BOSS关有特殊规则和挑战。

#### BOSS关特征

- **无升级卡奖励**：通过 BOSS 关后可获得一张**稀有卡牌**（只有 BOSS 关后才能得到）
- **特殊规则叠加**：每个 BOSS 关有1~2条特殊限制或机制

#### BOSS关变体（初版设计）

| BOSS关 | 名称 | 规则 | 挑战目标 |
|--------|------|------|---------|
| 第4关 | 哑弹危机 | 炸弹完全失效（无法放置） | 仅靠消行通关 |
| 第7关 | 巨兽来袭 | 出现1只超大史莱姆（3x3格，高HP，击杀高分） | 击败巨兽 + 消行 |
| 第10关 | 黑暗时代 | 棋盘每隔5秒随机熄灭2行（方块不可见） | 盲操生存 |
| 扩展 | 时间炸弹 | 棋盘底部有定时炸弹，到时自爆清除底部N行 | 赶在爆炸前消行 |
| 扩展 | 铁幕降临 | 每10块方块锁定就在底部插入一行硬化方块 | 防止棋盘溢出 |

#### 稀有卡牌（BOSS专属）

| 卡牌ID | 名称 | 正面效果 | 负面效果 |
|--------|------|---------|---------|
| chain_bomb | 连锁炸弹 | 炸弹爆炸会引爆相邻炸弹 | 炸弹自伤范围 +1格 |
| berserker | 狂战士 | HP越低，移动/跳跃速度越快（最高+50%） | 最大HP上限降至3 |
| gravity_flip | 重力反转 | 按住跳跃键可短暂贴附天花板行走 | 方块下落速度永久+20% |

### 4.8 分支关卡地图

游戏采用分支路线地图，玩家在部分节点可以选择走不同分支，形成每局独特的通关路径。

#### 地图结构

```
          [1] ──→ [2] ──→ [3]
                           │
                   ┌───────┴────────┐
                   ↓                ↓
                 [4 BOSS]        [4' 挑战]
                   │                │
                   └───────┬────────┘
                           ↓
                 [5] ──→ [6] ──→ [7]
                                  │
                         ┌────────┴────────┐
                         ↓                 ↓
                      [8 BOSS]         [8' 挑战]
                         │                 │
                         └────────┬────────┘
                                  ↓
                              [9] ──→ [10 BOSS]
```

#### 分支规则

- **BOSS 路**：正面面对 BOSS 关，通过后获得稀有卡牌奖励
- **挑战路**：跳过 BOSS，但面对加强版普通关（目标行数 ×1.5），只获得普通卡牌
- **分支选择**：在关卡完成的卡牌选择界面之后弹出路线选择UI
- **隐藏路**（扩展）：特定条件（如连续3关全程不扣血）解锁隐藏捷径

### 4.9 方块-角色联动

- 方块下落时检测与炸弹碰撞 → 引爆
- 方块锁定后检测角色碾压 → 伤害 + 推开
- 消行时角色脚下方块消失 → 角色自然下落
- 角色站在方块上 → 方块作为平台
- 特殊方块被消除/炸毁 → 触发对应奖励/惩罚逻辑（宝箱掉落、毒方块扣血等）

## 5. 操作方式

### 5.1 键盘（桌面）

| 按键 | 功能 |
|------|------|
| A / D | 角色左右移动 |
| Space | 角色跳跃 |
| E | 放置炸弹 |

| W | 确认放置（方块停止跟随） |
| S | 硬降 |

### 5.2 触屏（移动端）

**左侧 — 虚拟方向键**：
- 左右按钮控制角色移动
- 向上超过50%触发跳跃

**右侧 — 圆形按钮**（中文标签）：

| 按钮 | 功能 |
|------|------|
| 旋转 | 旋转方块 |
| 放置 | 确认方块位置 |
| 炸弹 | 放置炸弹 |
| 速降 | 硬降 |
| 暂停 | 暂停游戏 |

- 支持多点触控（最多4个触点）
- 按钮使用 consume 模式（每次按压只触发一次）

## 7. 游戏状态机

```
PLAYING ──pause──→ PAUSED ──unpause──→ PLAYING
   │
   ├──linesDetected──→ CLEARING_LINES
   │                        │
   │               ┌─animDone──→ PLAYING
   │               └─stageClear──→ STAGE_CLEAR
   │                                    │
   │                           ┌─普通关─→ cardSelected──→ routeSelected──→ PLAYING
   │                           └─BOSS关─→ bossCardSelected──→ routeSelected──→ PLAYING
   │
   └──characterDied / pieceOverlap──→ GAME_OVER
                                           │
                                      restart──→ PLAYING (回到关卡选择)
```

## 9. 游戏流程

```
标题画面（TitleScene）
  ↓ 按Space/点击
开始游戏（GameScene）
  ↓
生成新方块（跟随角色X位置）
  ↓
角色在棋盘内移动/跳跃/放炸弹
方块同时受重力下落
  ↓
玩家确认放置(W) 或 硬降(S)
  ↓
方块锁定 → 写入棋盘
  ├→ 检测角色碾压 → 扣血/推开
  ├→ 炸弹+1
  └→ 检测消行 → 计分
        ↓
    达到本关目标行数/分数？
    ├─ 是 → STAGE_CLEAR
    │         ├─ 普通关 → 升级卡选择（双面卡）→ 路线分叉选择 → 进入下一关
    │         └─ BOSS关 → 稀有卡选择 → 路线分叉选择 → 进入下一关
    └─ 否 → 继续游戏
  ↓
角色存活？ ──否──→ 游戏结束 → 按R重开
  │是
  ↓
返回"生成新方块"
```

## 10. 架构（模块说明）

**模块结构（~21 文件，~2600+ 行）：**

```
src/
  main.ts                        — Phaser 游戏配置（520x960竖屏，FIT缩放）
  constants.ts                   — 所有游戏参数（无导入依赖）
  events.ts                      — 事件名称常量（解耦用）
  TouchControls.ts               — 移动端触屏按钮覆盖层（脏标志渲染）
  utils/
    CollisionUtils.ts            — 共享 AABB/像素↔网格碰撞工具
  core/
    BoardModel.ts                — 棋盘数据模型（纯状态，无Phaser）
    TetrisEngine.ts              — 方块生命周期：生成/移动/旋转/锁定/重力/跟随
    CharacterPhysics.ts          — 角色物理：重力、碰撞/HP/伤害
    BombSystem.ts                — 炸弹放置/引信/爆炸/冲击波（无Phaser）
    GameStateMachine.ts          — FSM：PLAYING|PAUSED|CLEARING_LINES|STAGE_CLEAR|GAME_OVER
    LevelManager.ts              — 关卡进度管理：关卡目标行数、关卡推进
    PlayerUpgrades.ts            — 升级数据：6种强化类型、叠加逻辑
  input/
    InputManager.ts              — 键盘+触屏统一输入分发
  rendering/
    BoardRenderer.ts             — 棋盘绘制（脏标志优化）
    PieceRenderer.ts             — 活动方块 + 幽灵 + 连接线
    CharacterRenderer.ts         — 角色像素画 + HP/炸弹数量显示
    BombRenderer.ts              — 炸弹 + 爆炸视觉
    UIRenderer.ts                — 分数/关卡/消行/下一块预览/控制说明/游戏结束
    CardOverlay.ts               — 关卡完成升级卡选择界面
  scenes/
    TitleScene.ts                — 标题画面：装饰方块、操作说明、开始提示
    GameScene.ts                 — 薄编排层（创建子系统、连接事件、委托渲染）
```
